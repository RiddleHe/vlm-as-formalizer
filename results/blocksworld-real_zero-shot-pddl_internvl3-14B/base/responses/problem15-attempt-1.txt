To generate the PDDL problem file, we need to consider the goal which the robot must achieve and the current state of the blocks. Here’s how we write the PDDL problem file for this scenario:

```pddl
(define (problem blocksworld-problem)
    (:domain blocksworld)
    (:objects
        red block
        orange block
        purple block
        blue block
        yellow block
        green block
        robot1 robot
    )
    (:init
        (yellow clear)
        (green clear)
        (ontable yellow)
        (ontable green)
        (clear blue)
        (ontable blue)
        (on red blue)
        (clear orange)
        (on blue purple)
        (on purple orange)
        (handempty robot1)
    )
    (:goal 
        (and 
            (on yellow blue)
            (on blue purple)
            (on purple orange)
            (on orange green)
            (on green red)
        )
    )
)
```

### Explanation
- **Objects**: Enumerate all blocks and the robot.
- **Initial State**:
  - `yellow` and `green` are separate from the existing stack and are clear, placed on the table.
  - The existing stack: `(on red blue)`, `(on blue purple)`, `(on purple orange)`.
  - `robot1` is initially empty-handed.
- **Goal**:
  - The blocks should be stacked in the order `yellow -> blue -> purple -> orange -> green -> red`.

This PDDL problem captures the transformation needed from the current state to the desired goal by using the predefined actions (pick-up, put-down, stack, unstack) from the domain file.
