; Analysis of error:
; The last problem file had syntax errors mostly because of using "not" in the :init section in a way that is not allowed in PDDL. 
; PDDL initial state should only contain positive literals. We can't use (not (on_cutting_board carrot1)) etc. 
; Instead, we should simply omit those predicates if they are not true.
; Also, predicates like (cutting_board cutting_board1) need to be included since the domain uses it in preconditions but it was missing.
; I will fix these issues in the problem file.

(define (problem cooking1)
(:domain cooking)
(:objects
    robot1
    robot2
    bowl1 - bowl
    carrot1 tomato1 - vegetable
    knife1 - tool
    counter1 - counter
    cutting_board1 - cutting_board
)
(:init
    (at bowl1 counter1)
    (at carrot1 counter1)
    (at tomato1 counter1)
    (at cutting_board1 counter1)
    (free robot1)
    (holding robot2 knife1)
    (cutting_board cutting_board1)
)
(:goal 
  (and 
    (sliced carrot1)
    (sliced tomato1)
    (at carrot1 bowl1)
    (at tomato1 bowl1)
  )
)
)
