To create the PDDL problem based on the given instruction and the provided domain file, we need to define the objects, their initial states, and the goal specifications. Here's the PDDL file for the specified task:

```pddl
(define (problem blocksworld)
  (:domain blocksworld)
  (:objects
    yellow green pink red purple
  )
  (:init
    (clear yellow)
    (clear green)
    (clear pink)
    (clear red)
    (clear purple)
    (ontable yellow)
    (ontable green)
    (ontable pink)
    (ontable red)
    (ontable purple)
    (handempty robot)
  )
  (:goal
    (and
      (on yellow green)
      (on green pink)
      (on red purple)
    )
  )
)
```

### Explanation:
- **Objects**: We have five blocks: `yellow`, `green`, `pink`, `red`, and `purple`.
- **Initial State**:
  - Each block (`yellow`, `green`, `pink`, `red`, `purple`) is clear (`clear`).
  - Each block is initially on the table (`ontable`).
  - The robot's hand is empty (`handempty`).
- **Goal**:
  - There should be a stack of three blocks (`yellow` over `green` over `pink`).
  - There should be another stack of two blocks (`red` over `purple`).

This setup ensures that the blocks are arranged as per the given instruction.
