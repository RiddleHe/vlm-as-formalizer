To create a PDDL problem based on the given instruction and the provided domain file, we need to define the objects, their initial states, and the goal state. Here's the PDDL file for the specified task:

```pddl
(define (problem blocksworld_stack)
  (:domain blocksworld)
  (:objects
    orange - block
    red - block
    yellow - block
    green - block
    blue - block
    purple - block
  )
  (:init
    (clear orange)
    (ontable orange)
    (clear red)
    (ontable red)
    (clear yellow)
    (ontable yellow)
    (clear green)
    (ontable green)
    (clear blue)
    (ontable blue)
    (clear purple)
    (ontable purple)
    (handempty robot)
  )
  (:goal
    (and
      (on purple blue)
      (on blue green)
      (on green yellow)
      (on yellow red)
      (on red orange)
      (handempty robot)
    )
  )
)
```

### Explanation:
- **Objects**: We define six blocks (`orange`, `red`, `yellow`, `green`, `blue`, `purple`) as the objects that can be manipulated.
- **Initial State**:
  - Each block is initially clear and on the table.
  - The robot's hand is empty.
- **Goal State**:
  - The blocks are stacked in the order specified: purple over blue over green over yellow over red over orange.
  - The robot's hand remains empty after stacking the blocks.

This PDDL file sets up the problem correctly according to the given instruction and domain specifications.
