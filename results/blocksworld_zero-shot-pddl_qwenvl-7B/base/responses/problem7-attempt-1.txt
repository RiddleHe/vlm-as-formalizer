```pddl
(define (problem blocksworld-stack)
  (:domain blocksworld)
  (:objects
    yellow-block pink-block green-block red-block purple-block blue-block
  )
  (:init
    (clear yellow-block)
    (clear pink-block)
    (clear green-block)
    (clear red-block)
    (clear purple-block)
    (clear blue-block)
    (ontable yellow-block)
    (ontable pink-block)
    (ontable green-block)
    (ontable red-block)
    (ontable purple-block)
    (ontable blue-block)
    (handempty robot)
  )
  (:goal (and
          (on yellow-block pink-block)
          (on pink-block green-block)
          (on green-block red-block)
          (on red-block purple-block)
          (on purple-block blue-block)
          (handempty robot)
        )
  )
)
```

### Explanation:
- **Objects**: We have six blocks of different colors: yellow, pink, green, red, purple, and blue.
- **Initial State**:
  - Each block is clear (`clear` predicate).
  - Each block is on the table (`ontable` predicate).
  - The robot's hand is empty (`handempty` predicate).
- **Goal State**:
  - The blocks are stacked in the desired order: yellow over pink over green over red over purple over blue.
  - The robot's hand remains empty after stacking all the blocks.

This PDDL file sets up the problem correctly according to the given instruction and domain specifications.
