The error in the previous PDDL file is a semantic PDDL error. The predicate (at knife robot2) is invalid because the second argument of the predicate "at" must be of type location according to the domain definition, but robot2 is of type robot.

The correct way to represent that the robot is carrying the knife is only using the (carry ?robot ?obj) predicate. The object (knife) should not be located at a robot since robots are not locations.

(define (problem slice_tomatoes_and_place)
    (:domain cooking)
    (:objects
        tomato carrot - vegetable
        knife - tool
        bowl cutting_board counter - location
        robot1 robot2 - robot
    )
    (:init
        (available tomato)
        (available carrot)
        (available bowl)
        (available cutting_board)
        (is-whole tomato)
        (is-whole carrot)
        (free robot1)
        (carry robot2 knife)
        (can-cut knife)
        (at tomato counter)
        (at carrot cutting_board)
        (at bowl counter)
        (at cutting_board counter)
        (is-workspace cutting_board)
    )
    (:goal (and
        (is-sliced tomato)
        (at tomato bowl)
