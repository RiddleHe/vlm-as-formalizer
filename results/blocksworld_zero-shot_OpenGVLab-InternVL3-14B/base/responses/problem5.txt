```pddl
(define (problem blocksworld)
    (:domain blocksworld)
    (:objects
        yellow_block - block
        red_block - block
        blue_block - block
        orange_block - block
        green_block - block
        robot - robot
    )
    (:init
        (ontable yellow_block)
        (ontable red_block)
        (ontable blue_block)
        (ontable orange_block)
        (ontable green_block)
        (clear yellow_block)
        (clear red_block)
        (clear blue_block)
        (clear orange_block)
        (clear green_block)
        (handempty robot)
    )
    (:goal (and
        (on yellow_block red_block)
        (on red_block blue_block)
        (on blue_block orange_block)
        (on orange_block green_block)
        (clear yellow_block)
    ))
)
```

### Explanation:
1. **Objects**: Defined each block as an instance of `block` and the robot as an instance of `robot`.
2. **Initial State**:
   - All blocks are on the table (`ontable`).
   - Each block is clear (`clear`).
   - The robot is handempty (`handempty`).
3. **Goal**:
   - Define the desired stacking order with `on` predicates.
   - The top block in the stack should have no block on top of it (`clear yellow_block`).

This problem setup follows the constraints and actions defined in the domain, ensuring that the stacking can be achieved logically step-by-step.
