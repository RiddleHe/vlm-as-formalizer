; The error last time was that the knife was at robot1, but the robot was not carrying the knife. According to the domain, to slice, the robot must carry the tool (knife).
; Also, "at knife robot1" is not consistent with the domain's location predicate because "at" is for object-location, and robot1 is not declared as a location.
; Instead, the robot should be carrying the tool.
; Additionally, the initial state should specify which robot is carrying the knife (since the image shows one robot holding a knife).
; The cucumber, carrot, and tomato must be in a location that is a workspace for slicing.
; The bowl must be in a location accessible for placing ingredients.
; Adjusting these details should fix the problem.

(define (problem cooking_problem_2)
  (:domain cooking)
  (:objects 
    robot1 robot2 - robot
    knife spatula spoon - tool
    cucumber carrot tomato - vegetable
    bowl cutting_board counter - location
  )
  (:init 
    (free robot2) ; robot2 is free, robot1 holds the knife
    (carry robot1 knife)
    (not (free robot1))
    
    (available cucumber)
    (available carrot)
    (available tomato)
    
    (can-cut knife)
    
    (at cucumber cutting_board)
    (at carrot cutting_board)
    (at tomato cutting_board)
    (at bowl counter)
    
    (is-whole cucumber)
    (is-whole carrot)
    (is-whole tomato)
    
    (is-workspace cutting_board)
  )
  (:goal (and
    (is-sliced cucumber)
    (is-sliced carrot)
    (is-sliced tomato)
    (at cucumber bowl)
    (at carrot bowl)
    (at tomato bowl)
  ))
