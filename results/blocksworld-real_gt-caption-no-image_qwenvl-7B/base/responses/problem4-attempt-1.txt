To solve the given problem using the `blocksworld` domain, we need to define the initial state, the goal state, and the actions that can be performed by the robot.

### Step-by-Step Solution:

1. **Define Objects**: We have blocks of different colors (orange, red, purple, blue, yellow, green).
2. **Initial State**: The initial state specifies the arrangement of the blocks.
3. **Goal State**: The goal state specifies the desired arrangement of the blocks into two stacks.

### PDDL Problem Definition:

```pddl
(define (problem blocksworld)
    (:domain blocksworld)
    (:objects
        robot1 - robot
        orange - block
        red - block
        purple - block
        blue - block
        yellow - block
        green - block
        table - table
    )
    (:init
        (clear orange)
        (clear red)
        (clear purple)
        (clear blue)
        (clear yellow)
        (clear green)
        (ontable orange)
        (ontable red)
        (ontable purple)
        (ontable blue)
        (ontable yellow)
        (ontable green)
        (handempty robot1)
    )
    (:goal
        (and
            (on purple green)
            (on blue red orange yellow)
        )
    )
)
```

### Explanation:

1. **Objects**:
   - `robot1`: The robot arm.
   - `orange`, `red`, `purple`, `blue`, `yellow`, `green`: Blocks of different colors.
   - `table`: The surface where the blocks are initially placed.

2. **Initial State**:
   - All blocks (`orange`, `red`, `purple`, `blue`, `yellow`, `green`) are clear and on the table.
   - The robot arm (`robot1`) is empty-handed.

3. **Goal State**:
   - Two stacks are created:
     - The first stack consists of `purple` on top of `green`.
     - The second stack consists of `blue` on top of `red` on top of `orange` on top of `yellow`.
