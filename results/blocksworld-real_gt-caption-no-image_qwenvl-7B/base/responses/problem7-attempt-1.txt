To solve the given problem using the `blocksworld` domain, we need to define the initial state, the goal state, and the actions that can be performed by the robot.

### Initial State
- The robot (`robot1`) is on the table and empty-handed (`handempty`).
- There is one stack of blocks (`orange`, `green`, `red`, `purple`, `yellow`, `blue`) on the table, arranged as specified in the instruction.

### Goal State
- The goal is to have the blocks stacked in the order: `blue`, `orange`, `green`, `purple`, `yellow`, `red`.

### Actions
The available actions from the domain file are:
- `pick-up`: Picks up a block from the table if it's clear and the robot is empty-handed.
- `put-down`: Puts down a block onto the table if the robot is holding a block.
- `stack`: Stacks a block on top of another block if the robot is holding a block and the target block is clear.
- `unstack`: Unstacks a block from another block if the robot is empty-handed and the block is on top of another block.

### PDDL Problem

```pddl
(define (problem blocksworld-stack)
  (:domain blocksworld)
  (:objects
    robot1 - robot
    table - table
    orange - block
    green - block
    red - block
    purple - block
    yellow - block
    blue - block
  )
  (:init
    (clear orange)
    (clear green)
    (clear red)
    (clear purple)
    (clear yellow)
    (clear blue)
    (ontable orange)
    (ontable green)
    (ontable red)
    (ontable purple)
    (ontable yellow)
    (ontable blue)
    (handempty robot1)
    (on table robot1)
  )
  (:goal
    (and
      (clear blue)
      (clear orange)
      (clear green)
      (clear purple)
      (clear yellow)
      (on blue orange)
      (on orange green)
      (on green red)
      (on red purple)
      (on purple yellow)
      (on yellow blue)
    )
  )
)
```

### Explanation
- **Objects**: We define the robot (`robot1`), the table, and the six blocks (`orange`, `green`, `red`, `purple`, `yellow`, `blue`).
- **Init**: The initial state specifies that each block is clear and on the table, and the robot is empty-handed and on the table.
- **Goal**: The goal is to have the blocks stacked in the desired order: `blue` on top of `orange`, `orange` on top of `green`, and so on.

This PDDL file correctly represents the initial state, the goal state, and adheres to the constraints defined in the `blocksworld` domain.
