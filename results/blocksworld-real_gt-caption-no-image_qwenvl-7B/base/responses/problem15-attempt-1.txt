To solve the given problem using the `blocksworld` domain, we need to define the initial state, the goal state, and the actions that can be performed by the robot.

### Step-by-Step Solution:

1. **Define Objects**: We need to define the blocks and their colors.
2. **Initial State**: Define the initial positions and states of the blocks and the robot.
3. **Goal State**: Define the desired final configuration of the blocks.
4. **Actions**: Use the available actions (`pick-up`, `put-down`, `stack`, `unstack`) to move the blocks from one place to another.

### PDDL File

```pddl
(define (problem blocksworld)
    (:domain blocksworld)
    (:objects
        robot1 - robot
        table - table
        green - block
        yellow - block
        red - block
        blue - block
        purple - block
        orange - block
    )
    (:init
        (clear green)
        (clear yellow)
        (clear red)
        (clear blue)
        (clear purple)
        (clear orange)
        (ontable green)
        (ontable yellow)
        (ontable red)
        (ontable blue)
        (ontable purple)
        (ontable orange)
        (handempty robot1)
    )
    (:goal
        (and
            (clear orange)
            (clear purple)
            (clear blue)
            (clear yellow)
            (clear green)
            (clear red)
            (ontable orange)
            (ontable purple)
            (ontable blue)
            (ontable yellow)
            (ontable green)
            (ontable red)
            (holding orange)
            (holding purple)
            (holding blue)
            (holding yellow)
            (holding green)
            (holding red)
            (on orange purple)
            (on purple blue)
            (on blue yellow)
            (on yellow green)
            (on green red)
        )
    )
)
```

### Explanation:

1. **Objects**:
   - `robot1`: The robot arm.
   - `table`: The surface where the blocks are initially placed.
   - `green`, `yellow`, `red`, `blue`, `purple`, `orange`: The blocks with their respective colors.

2. **Initial State**:
   - All blocks (`green`, `yellow`, `red`, `blue`, `purple`, `orange`) are on the table.
   - The robot (`robot1`) is empty-handed (`handempty`).

3. **Goal State**:
   - The blocks are stacked in the desired order: `yellow` over `blue` over `purple` over `orange` over `green` over `red`.
   - Each block is clear (`clear`) except for the top block in each stack.
   - The robot holds each block in the correct order as it moves them to the final position.
