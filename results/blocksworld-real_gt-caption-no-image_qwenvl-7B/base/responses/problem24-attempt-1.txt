### Step 1: Define Objects
We have three stacks of blocks, each containing multiple blocks. We'll use the following blocks:
- Orange (orange1, orange2)
- Purple (purple1, purple2, purple3)
- Green (green1, green2)
- Blue (blue1)
- Yellow (yellow1)
- Red (red1)

### Step 2: Initial State
The initial state will be as described in the instruction:
- Stack 1: orange1 on orange2 on table
- Stack 2: purple1 on purple2 on purple3 on table
- Stack 3: yellow1 on red1 on table

### Step 3: Goal State
The goal state is to have a single stack with the blocks arranged as follows:
- Blue on top
- Orange on second from the top
- Green on third from the top
- Purple on fourth from the top
- Red on fifth from the top
- Yellow at the bottom

### Step 4: Generate the PDDL File

```pddl
(define (problem blocksworld)
    (:domain blocksworld)
    (:objects
        orange1 orange2
        purple1 purple2 purple3
        green1 green2
        blue1
        yellow1
        red1
        robot1
        table
    )
    (:init
        (clear orange1)
        (clear orange2)
        (clear purple1)
        (clear purple2)
        (clear purple3)
        (clear green1)
        (clear green2)
        (clear blue1)
        (clear yellow1)
        (clear red1)
        (ontable orange1)
        (ontable orange2)
        (ontable purple1)
        (ontable purple2)
        (ontable purple3)
        (ontable green1)
        (ontable green2)
        (ontable blue1)
        (ontable yellow1)
        (ontable red1)
        (handempty robot1)
    )
    (:goal
        (and
            (on blue1 orange1)
            (on orange1 green1)
            (on green1 purple1)
            (on purple1 purple2)
            (on purple2 purple3)
            (on purple3 red1)
            (on red1 yellow1)
            (clear blue1)
            (clear orange1)
            (clear green1)
            (clear purple1)
            (clear purple2)
            (clear purple3)
            (clear red1)
            (clear yellow1)
            (handempty robot1)
        )
    )
)
```

### Explanation:
- **Objects**: We define all the blocks and the robot arm `robot1` and the table `table`.
- **Init**: The initial state is set up according to the instruction. Each block is clear and on the table.
- **Goal**: The goal is to have the blocks stacked in the specified order with the robot arm empty-handed.

This PDDL file correctly represents the problem as per the given domain and instruction.
